System Requirements

    Operating Systems:

    • Linux 2.4.20 kernel
    • Linux 2.6.22 kernel
    • VxWorks Tornado 2.1(END)

    • BCM94705LGR, BCM94704agr, BCM94704mpcb  -or-
    • Base Reference Design with a BCM94321 MiniPCI card -or-
    • Base Reference Design with a BCM94322 MiniPCI card -or-
    • Base Reference Design with a BCM94312 SDIO module -or-
    • Vendor Equivalent

What's New in Release 5.22.66?

  • Based on wl driver version 5.10.
  • Fix for ACI detection for AP: do not condition on association state (as done for a STA)
  • Fix two issues relating to AP Interference Mitigation when Autochannel is off
  • Better checking for sprom/OTP availability and fallback to nvram
  • CFE: Fixed boundary check error in lzma decompression.
  • Recognize 43111/43112 devices
  • Fix to allow the pci probe function to accept the device ids 43xxx

What's New in Release 5.22.65?

  • Based on wl driver version 5.10.119
  • Fix to mute the phy if 11h is enabled and we are AP operating on radar sensitive channel.

What's New in Release 5.22.64?

  • Based on wl driver version 5.10.
  • By default restrict TKIP associations from 11n STA's
  • Fix TGN 4.2.40: Beacon does not advertise RIFS as 0
  • DFS Update: changed radar args npulses to 4 ncontig to 2 for EU based locales. This applies to both 20MHz and 40MHz radarargs.
  • DFS Update: Added enabling filters for FAA radars, pw=2us

What's New in Release 5.22.61?

  • Based on wl driver version 5.10.114
  • Reject the assoc/reassoc request if a 11n STA tries to connect using TKIP encryption
  • Fix hang/crash issue observed on 4718nr+4322 and 4718nrl+43224.
  • Changes to support TGN Test 4.2.39, Transmit STBC/Non-STBC frames to STBC/Non-STBC capable STA respectively
  • Add counter for STBC and SGI tx/rx

What's New in Release 5.22.60?

  • Based on wl driver version 5.10.111
  • Expose OBSS coex control on external UI for router
  • AP now checks for any 40 intolerant association before upgrade to 40MHz
  • Add wl command to mask the trigger event A for obss support to support test
  • Fix router reboots during frequency accuracy test
  • AP Autochannel Interference Mitigation fix for legacy mode
  • Performance improvements for encrypted WET links
  • Make the max number of BSSIDs under hardware MBSSID programmable (but power of 2).
  • Add ucode based CCA stats
  • Change ISRAEL Locale settings for 2.4GHz operation (802.11n - CDD/SDM) to Locale Bn
  • Changes for CLMv2.6 and CLMv2.6.1
  • Add Rx Chain Powersave mode
  • Fix cannot send BAR when out of memory
  • Fix radio register VCO CAL settings for 4322, 43222, 43224

What's New in Release 5.22.54?

  • Based on wl driver version 5.10.105
  • Better IPA calibration in presence of WMM traffic
  • Remove the restriction of not transmitting probe responses during pre-tbtt when hw MBSSID is enabled.

What's New in Release 5.22.52?

  • Based on wl driver version 5.10.102
  • Improvements to interference mitigation
  • Major ACI/noise mitigation restructuring
  • CLM update for 43224/43225
  • Fixes for the issue where beacons are sent on secondary channel as a result of Autochannel selection
  • Rate selection fixes for handling of fallback rates
  • Fix 4321 RvR 2.4G @20 sensitivity issue
  • CLM 2.3 for 471x based designs
  • Fix an old bug that when ON time is smaller than OFF time, LED will never be turned on.
  • fallback rate retransmissions for non-AMPDUs is broken (ucode would use the primary rate instead of the fallback)
  • Need to report short GI capability on chipsets that support it
  • Work-arounds to squash the 2440 MHz spurs in 40 MHz channels on 43222MP2 boards with shields when the 2G band spur WAR boardflag is enabled.
  • Fix bug in idle TSSI initialization
  • Fix 802.1d country_ie interpretation for 5G channels
  • Take care of the cases that LED timer fires earlier than expected.
  • Add AP Interference Mitigation support
  • Fix bug in wlc_process_beacon
  • Fix a problem with the duty cycle for mimo frames (they would have used the duty cycle value of cck frames as opposed to ofdm).
  • Fixes for wl interference mode 4
  • Support reading OTP on PCI based chips
  • Enable 2o3 antenna selection for 4322x

What's New in Release 5.22.41?

  • Based on wl driver version 5.10.85
  • Fix WiFi TGN test 4.2.22 (step10) failure
  • Regulatory update to add ch14 cck to locale A2
  • Use lzma compression for compressed CFE
  • For newer MIMO chips, limit transmission of CCK packets to Ant0 only, based on board flags2
  • Changed bridge port allocation flag to ATOMIC to fix router crashes in some tests
  • Enhancements to the display of the "wl curpower" command
  • DFS false detection improvements
  • Fix HT40 mode high channels cannot reach full power
  • Performance enhancements for encrypted traffic
  • Rate selection improvements

What's New in Release 5.22.39?

  • Based on wl driver version 5.10.79
  • WPS fix for Vista Router failing test "WCN Error Handling and Temp Test 4"
  • Enable IPv6 for Vista Router builds
  • Fix WDS with AES encryption

What's New in Release 5.22.34?

  • Based on wl driver version 5.10.79
  • Fix for crash when using packet engine in mfg tests
  • DFS/Radar improvements
  • Check the SSID in association requests from a client STA
  • Send cts-to-self to prevent calibration being disrupted
  • Fix transmission of fallback rates for AMPDU
  • New regulatory Country/Locale Matrix v2.1, v2.1.1

What's New in Release 5.22.27?

  • Based on wl driver version 5.10.67
  • EVM improvements, especially for 43222 5GHz band
  • Fix accounting of rx crs glitch count and bphy glitch count

What's New in Release 5.22.26?

  • Based on wl driver version 5.10.65
  • Initial router release with support for bcm43222
